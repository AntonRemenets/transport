Добро пожаловать на страничку моего проекта Путник!

Этот учебный проект создан на фреймворке NestJS и несет цель демонстацию навыков программированиия серверной части программ. Принципиальная схема приложения тут.

Стек приложения.

  • интерфесы - RestApi, GraphQL
  • базы данных - Postgres, MongoDB, Redis
  • orm - Prisma, Mongoose
  • авторизация и аутентификация приближена к реальной - JWT, PassportJS
  • валидация - class-validator
  • раздача статичной верстки средствами ServeStatic
  • Nginx - куда ж без него

Эндпойнты.

Для REST API:

  • /auth/register - регистрация нового пользователя. На вход ожидается обьект с полями email и password
  • /auth/login - на вход ожидается обьект с полями email и password
  • /auth/logout
  • /auth/refresh-tokens - обновление токенов
  • кроме того существуют закрытые роуты для администротора

Для GraphQL эндпойнт - /graphql. В браузере зайти не получится (playground = false). Схема gql для постмана.Не забыть указать токен authorization, тип bearer. Токен выдается при логине.

Базы данных.

Используются 3 разных базы:

  • PostgreSQL в качестве основной базы данных. В ней хранятся данные о маршрутах, автобусах и водителях. Схему базы данных можно найти тут.
  • MongoDB используется для хранения регистрационных данных пользователей, их токенов и айпи адресов.
  • Redis используется для кеширования
